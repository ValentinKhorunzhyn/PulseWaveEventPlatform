🛠️ Pulse Wave Event Platform - приложение для генерации и регистрации событий. Событийно - ориентированная архитектура.
Построено на:
- Java 25
- Spring Boot 4+
- Spring Framework 7
- Spring Data JPA + Hibernate
- Spring Security (Keycloak)
- Lombok
- FlyWay
- Apache Kafka
- Debezium
- PostgreSQL
- Micrometer
- ELK (Elasticsearch - LogStash - Kibana)
- Prometheus/Grafana
- Jaeger
- Docker
- Maven

🧩 Обзор  
1. Бэкенд система, позволяющая пользователям управлять данными событий с полной поддержкой CRUD-функций.  
2. 3 сервиса Publisher генерируют случайные события (USER_ACTION, SYSTEM_ALERT, BUSINESS_EVENT, AUDIT_LOG),
сохраняют данные события в 
общую базу данных в 2 разные таблицы (table_events and table_outbox_events) в рамках одной транзакции
3. Сервис Kafka-Connect (Debezium) читает данные WAL журнала базы данных и все новые зарегистрированные ивенты отправляет
в брокера сообщений в специальный топик в формате Avro.
4. Сервис Registar принимает информацию от брокера и сохраняет событие в своей базе данных. Затем регистратор обрабатывает
событие с учетом бизнес логики и отмечает событие как обработанное или нет.
5. В сервисе Registar специальная задача один раз в несколько секунд достает из базы обработанные события и отправляет их 
обратно в брокера сообщений и при подтверждении получения от брокера помечает событие в базе данных как отправленное
6. Сервис Publisher читает данные из брокера о подтвержденных событиях и отмечает у себя в базе данных событие как
подтвержденное

На стороне Registar реализована проверка на дубликаты входящих сообщений, когда сообщение успешно обработано - оно
сохраняется в базу Registar. Между Publisher и Registar все события передаются асинхронно в формате Avro

📦 Как запустить  
Через Docker:  
В корне проекта в терминале выполнить команду:  
docker-compose down -v  
docker-compose up -d --build  

Доступ к сервисам:  
http://localhost:8085 — Api Gateway. (for Publishers)  
http://localhost:8091 — Publisher service 1.  
http://localhost:8092 — Publisher service 2.  
http://localhost:8093 — Publisher service 3.  
http://localhost:8094 — Registar service.  
http://localhost:8080 — Keycloak.  
http://localhost:9092 — Apache Kafka.  
http://localhost:5601 — Kibana.  
http://localhost:3000 — Grafana.  

Как настроить Kibana: server --> Discover --> Create new data view --> index pattern (microservices-logs-*) --> save  
Как настроить Grafana:
1. Sign in Grafana UI (http://localhost:3000 ) with login and password (admin, admin)
2. Navigate to menu: connections --> data sources --> add datasource --> Prometheus --> 
Prometheus server url (http://prometheus:9090) --> set Save&Test
3. Navigate to menu: dashboards --> create dashboard --> import dashboard --> and import your favorite dashboard from 
local storage or Grafana server using dashboard ID (for example: 22108). You can find example json in project's
observability folder

📁Примечание:
- Топик в брокере для получения сообщений от генератора сразу состоит из 3 партиций поскольку генераторы отправляют 
событие по определенному ключу и брокер корректно распределяет сообщения.
- На стороне регистратора работают одновременно 3 потока чтения событий из брокера сообщений
- В рамках тестового задания сервисы и мапперы не имплементят интерфейсы, чтобы не нагромождать файлами структуру проекта
- В регистраторе реализован глобальный обработчик ошибок
- Тестировать API лучше через Postman -> импортировать JSON сваггера как коллекцию (Swagger часто зависает)
